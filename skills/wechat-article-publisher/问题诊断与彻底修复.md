# 微信文章乱码问题 - 彻底修复方案

**问题发现时间**: 2026-03-19 20:35  
**问题现象**: 微信草稿箱文章显示 `\u4e2d\u533b\u98df\u7597\u517b\u751f\u65b9`（Unicode 转义字符）  
**根本原因**: `requests.post(json=data)` 默认使用 `ensure_ascii=True`  

---

## 🔍 问题根源分析

### 错误代码模式
```python
# ❌ 错误方式 - 会导致乱码
response = requests.post(url, params=params, json=data)
# requests 内部会自动调用 json.dumps(data, ensure_ascii=True)
# 中文被转换为 \uXXXX 格式
```

### 正确代码模式
```python
# ✅ 正确方式 - 无乱码
json_data = json.dumps(data, ensure_ascii=False)
response = requests.post(
    url, 
    params=params, 
    data=json_data.encode('utf-8'),
    headers={'Content-Type': 'application/json; charset=utf-8'}
)
# 手动控制 JSON 编码，确保使用 UTF-8
```
