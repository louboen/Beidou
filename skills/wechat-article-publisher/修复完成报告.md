## 🎯 问题总结

### 问题现象
微信草稿箱文章显示 Unicode 转义字符：
```
\u4e2d\u533b\u98df\u7597\u517b\u751f\u65b9
```
应该显示：
```
中医食疗养生方
```

### 根本原因
**`requests.post(json=data)` 默认使用 `ensure_ascii=True`**

当使用 `json=` 参数时，requests 库会自动调用：
```python
json.dumps(data, ensure_ascii=True)  # ❌ 默认值
```

这会将中文转换为 Unicode 转义字符 `\uXXXX` 格式。

## 🔧 核心修复要点

### 1. 禁止使用 `json=` 参数
```python
# ❌ 禁止使用
response = requests.post(url, json=data)

# ✅ 必须使用
json_data = json.dumps(data, ensure_ascii=False)
response = requests.post(url, data=json_data.encode('utf-8'))
```

### 2. 必须使用 `ensure_ascii=False`
```python
# ❌ 错误（默认）
json_bad = json.dumps(data)  # ensure_ascii=True

# ✅ 正确
json_good = json.dumps(data, ensure_ascii=False)
```

### 3. 手动编码为 UTF-8
```python
# ✅ 正确
data_bytes = json_data.encode('utf-8')
```

### 4. 设置正确的 Content-Type
```python
# ✅ 正确
headers = {'Content-Type': 'application/json; charset=utf-8'}
```
