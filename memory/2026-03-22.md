## 🔧 Win11 SSH 远程连接配置（20:07 - 21:43）

### 任务背景
- **目标**: 通过 SSH 连接到 Win11 设备 (BOEN-BOOK)
- **网络**: Tailscale 组网 (`100.64.0.2`)
- **账户**: `loubo` (Microsoft 账户，无密码)

### 执行步骤

#### 1. 创建 SSH 启用脚本 (20:17)
- **文件**: `scripts/enable-ssh-win11.ps1`
- **功能**: 自动安装/启用 OpenSSH Server
- **上传**:
  - GitHub: https://github.com/louboen/openclaw-backup/blob/main/scripts/enable-ssh-win11.ps1
  - 飞书文档：https://feishu.cn/docx/VfaFdianiotaW3xgfG4cCmzInih

#### 2. Win11 端配置
```powershell
# 执行脚本启用 SSH
.\enable-ssh-win11.ps1

# 生成 SSH 密钥
ssh-keygen -t ed25519 -f "$env:USERPROFILE\.ssh\id_ed25519" -N ""

# 添加公钥到系统级 authorized_keys
Add-Content -Path "C:\ProgramData\ssh\administrators_authorized_keys" -Value $pubKey
```

#### 3. 关键问题解决

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| SSH 服务未安装 | Windows 默认不安装 | PowerShell 脚本自动安装 |
| 公钥认证失败 | 权限配置错误 | 使用 `administrators_authorized_keys` 而非用户目录 |
| 权限拒绝 | ACL 权限不正确 | 只允许 SYSTEM 和 Administrators 组 |
| 空密码登录失败 | Windows OpenSSH 禁止空密码 | 使用 SSH 密钥认证（最终方案） |

#### 4. 成功配置
- **服务器端**: `/home/admin/.ssh/authorized_keys` 添加 Win11 公钥
- **Win11 端**: `C:\ProgramData\ssh\administrators_authorized_keys` 添加服务器公钥
- **密钥文件**: `/tmp/win11-ssh-key` (临时，需长期保存)

### 最终连接命令
```bash
ssh -i /tmp/win11-ssh-key loubo@100.64.0.2
```

### 最佳实践总结

#### Windows SSH 配置要点
1. **管理员账户** 必须使用 `C:\ProgramData\ssh\administrators_authorized_keys`
2. **权限要求** 极其严格：只能有 SYSTEM 和 Administrators 组权限
3. **空密码登录** Windows OpenSSH 默认禁止，建议用密钥认证
4. **Microsoft 账户** 不能用 `net user` 修改密码

#### SSH 密钥认证流程
```
1. 服务器生成密钥对 (ssh-keygen)
2. 公钥添加到 Win11 administrators_authorized_keys
3. 设置严格权限 (SYSTEM + Administrators only)
4. 重启 SSH 服务
5. 测试连接 (ssh -i <私钥> user@host)
```

#### 文件权限设置（PowerShell）
```powershell
$acl = New-Object System.Security.AccessControl.FileSecurity
$acl.SetAccessRuleProtection($true, $false)
$acl.AddAccessRule((New-Object System.Security.AccessControl.FileSystemAccessRule("SYSTEM", "FullControl", "Allow")))
$acl.AddAccessRule((New-Object System.Security.AccessControl.FileSystemAccessRule("Administrators", "FullControl", "Allow")))
Set-Acl -Path $authFile -AclObject $acl
```
